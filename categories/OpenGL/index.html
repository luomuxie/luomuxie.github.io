LearnOpenGL16-ShadownMapping

引言

阴影的重要性

阴影在3D图形中扮演着至关重要的角色，因为它们增加了场景的深度和立体感，同时为用户提供了关于物体空间关系和位置的线索。没有阴影的3D场景通常看起来会非常平面和不真实。阴影可以帮助改善物体的识别率，增强视觉效果，并且为虚拟世界带来更多的真实感和沉浸感。

shadow mapping 的基本概念

阴影映射是一种在计算机图形中生成阴影的技术。这种技术的基本思想是从光源的视角渲染场景，从而创建一个深度图，这个深度图包含了从光源视角看到的最近表面的深度信息。然后，在正常渲染过程中，场景中的每个点都会被转换到光源的视角，以确定它是否被阻挡在光源后面。

如上图，左侧的图片展示了一个定向光源（所有光线都是平行的）在立方体下的表面投射的阴影。通过储存到（右边）深度贴图中的深度值，我们就能找到最近点，用以决定片段是否在阴影中。

基本原理

光源视角 (Light Space )

光源视角是指从光源的位置出发，按照光的方向查看场景的视角。在这个视角下，光源可以被看作是一个“观察者”，它可以“看到”所有直接被照亮的表面。这种视角对于确定哪些部分会被光照到，哪些部分会形成阴影至关重要。

深度图 (Depth Map)

深度图，也称为阴影图，是一个纹理，记录了从光源视角看到的每个像素点的深度信息。它是通过渲染场景的深度信息到一个离屏缓冲区（off-screen buffer）来创建的，并且只存储深度值，不存储颜色信息。

如何从光源视角渲染深度图

  • 第1步：设置光源的视图和投影矩阵,

首先，你需要定义光源的位置和方向。根据这些信息，你可以创建一个视图矩阵，它能够将场景中的坐标从世界空间变换到光源空间。这个过程与设置相机的视图矩阵非常相似。接着，你需要设置一个投影矩阵，这取决于你的光源类型（点光源、定向光源或聚光灯）。对于定向光源，你通常会使用一个正交投影矩阵；对于点光源或聚光灯，你会使用一个透视投影矩阵。

  • 第2步：创建并设置深度缓冲区:在OpenGL中，你需要创建一个帧缓冲区对象（FBO），并附加一个深度纹理。这个深度纹理将作为深度缓冲区，用来存储深度信息。

  • 第3步：渲染场景到深度缓冲区:

    • 清除深度缓冲区：在每次渲染之前，清除深度缓冲区，通常设置深度值为最远。

    • 使用光源的视图和投影矩阵：为渲染设置适当的矩阵，这样所有顶点都会从光源的视角被变换。

    • 只写入深度信息：渲染场景时，你只关心深度信息，不需要颜色信息。

    • 遍历场景中的物体：对每个物体，应用视图和投影矩阵变换，并渲染它们的深度信息到深度纹理中。

  • 第4步：使用深度图进行阴影判断

    一旦深度图被渲染，它就可以被用来在实际的场景渲染中判断片元是否处于阴影之中。这个过程涉及将片元的世界空间坐标变换到光源空间，并与深度图中存储的值进行比较

阴影映射的挑战

深度冲突（Shadow Acne）

使用平行光源（如太阳）时。这种情况下，由于深度贴图的分辨率限制以及视角的问题，多个不同的片段（fragment）可能会从深度贴图的同一像素进行深度值的采样。当光源以一定角度照射表面时，这个问题尤为明显。

就像下图中，多个不同的片段（fragment）可能会从深度贴图的同一像素进行深度值的采样，a,b,c d 三个点，共同从中心的红点采样，c ,d 深度明显大于红点深度，a，b的深度小于红点深度

阴影偏移（shadow bias）

对深度图中存储的深度值添加一个小的偏差值。这个偏差通常是一个非常小的正数，目的是轻微地提高深度图中的深度值。这样，当我们从相机的视角查看场景并进行深度比较时，由于深度图中的值略微增加，那些本应照亮的片元不太可能会被错误地判断为在阴影中，因为它们现在与深度图中存储的略微偏差的值相比较了。

但是有些表面坡度很大，仍然会产生阴影失真，如下图：

在表面坡度较大的情况下，即表面与光线方向接近平行，即使相邻的片元之间的实际距离很小，由于深度的变化与视线方向的变化不成正比，它们在深度图中的深度值的变化却可能很大。

这意味着即使实际的几何表面没有变化，深度值的微小差异也可能导致深度冲突，特别是在表面与光线方向几乎平行的情况下。

添加分辨率增多采样

提高阴影映射中的深度图（或阴影贴图）分辨率是减轻或消除阴影痤疮（Shadow Acne）的一种方法。阴影痤疮是由于深度贴图的分辨率不足以及深度缓冲的有限精度所引起的，表现为渲染的阴影表面上出现的不希望的条纹状或斑点状伪影。

提高分辨率意味着在阴影贴图中每个阴影像素可以覆盖更小的区域，这可以减少不正确阴影的出现，提供更精细的阴影边缘和细节。

限制：

  1. 性能开销：提高阴影贴图的分辨率会增加内存的使用和渲染计算的负担。在实时渲染的应用中，如视频游戏，这可能会影响帧率和总体性能。
  2. 可能的过滤问题：即使提高了分辨率，阴影边缘可能仍然出现锯齿状或过于硬边的问题，这可能需要通过软阴影技术如百分比较近过滤（PCF）来进一步处理。
  3. 存储和带宽限制：更高分辨率的阴影贴图需要更多的内存存储空间，同时也增加了GPU带宽的需求。

悬浮阴影（Peter Panning）

阴影偏移是为了解决深度冲突（Shadow Acne）问题而添加的。深度冲突是由于深度缓冲区的有限精度导致的，在渲染时可能会产生阴影表面上的斑点状伪影。为了避免这种情况，通常会向深度值添加一个小的偏移量，从而确保深度测试不会因为微小的误差而失败。

然而，如果偏移量设置得过大，就会产生Peter Panning效应，因为实际的阴影会从物体表面偏移开来，看起来像是悬浮在空中的。正确的偏移量通常是一个需要精心调整的值，它需要足够大以避免深度冲突，但又不能太大以至于产生不真实的阴影间隙。

百分比较近过滤（PCF）

基础PCF：

最简单的PCF实现。它在深度贴图上对一个小的区域（例如3x3或5x5像素网格）进行采样，并对这些样本进行平均，来决定最终像素是否在阴影中。

uniformDiskSample PCF

在圆盘内随机或伪随机选择采样点，以模拟光源的面积和扩散效果

poisson disk PCF

它通过采用特定的采样策略来改善阴影质量。Poisson Disk采样是一种在给定区域内生成样本点的方法，这些样本点既是随机分布的，又保持一定的最小距离，从而避免了样本点的聚集。

更均匀的采样分布：相比于简单的随机或均匀采样，Poisson Disk采样在样本点之间保持最小距离，这避免了样本点的过度聚集或规则排列，从而产生更自然的阴影效果。

方差阴影映射（VSM）（TODO）

VSM 主要依赖于统计学中的方差概念来实现阴影的软化效果。

  1. 存储深度信息的方式：
    • 传统阴影映射技术通常只存储深度信息。相比之下，VSM 存储每个像素的深度信息和深度的平方。
    • 存储额外的深度平方信息允许计算出该像素深度值的方差。
  2. 使用方差进行阴影判断：
    • 在渲染过程中，VSM 使用这些存储的值来计算阴影区域内的方差和平均深度。
    • 利用这些信息，VSM 可以估算一个像素是否处于阴影中，而不是进行简单的深度比较。
  3. 软阴影效果：
    • 方差的使用允许阴影边缘出现渐变，这样阴影边缘就不是硬边缘而是软边缘，能够生成更自然的阴影效果。
    • 这种方法特别适用于光源较大或阴影边缘需要更柔和的场景。
  4. 优点：
    • VSM 可以生成非常平滑的阴影边缘，并且处理起来相对简单。
    • 它不像传统的PCF那样依赖于大量的采样来软化阴影边缘，因此在某些情况下可能更高效。
  5. 挑战和限制：
    • VSM 可能会遇到光泄漏（light leaking）和阴影穿透（shadow bleeding）的问题，特别是在复杂的场景或重叠几何体的情况下。
    • 它对深度值的分布非常敏感，可能需要进行调整和优化，以避免这些问题。

级联阴影映射（CSM）(TODO)

级联阴影映射（Cascade Shadow Maps，简称CSM）是一种用于实现大范围场景阴影的技术，特别是在户外环境和大型3D场景中。CSM 通过分割视图摄像机的视锥体（frustum）到多个区域（称为“级联”），为每个区域生成单独的阴影贴图，从而在不同的距离级别上提供不同的阴影细节。（LOD思想）

级联阴影映射（Cascade Shadow Maps，简称CSM）是一种用于实现大范围场景阴影的技术，特别是在户外环境和大型3D场景中。CSM 通过分割视图摄像机的视锥体（frustum）到多个区域（称为“级联”），为每个区域生成单独的阴影贴图，从而在不同的距离级别上提供不同的阴影细节。

CSM的主要特点和工作原理：

  1. 分割视图摄像机的视锥体：
    • CSM 将摄像机的视锥体分割成多个级联，通常是2至4个。每个级联覆盖视锥体的不同部分，通常近处的级联覆盖区域更小，但使用更高分辨率的阴影贴图。
  2. 为每个级联生成阴影贴图：
    • 对于每个级联，分别从光源的视角渲染阴影贴图。这意味着每个级联都有自己的深度贴图，能够独立控制其细节水平。
  3. 提供详细的近处阴影和合理的远处阴影：
    • 由于近处的级联使用更高分辨率的阴影贴图，它们能够提供更精细的阴影细节。而远处的级联则可以使用较低分辨率的贴图，减少性能消耗。
  4. 平滑过渡：
    • CSM 需要在不同级联之间平滑过渡，以避免可见的切换或阴影不连续的问题。
  5. 优点：
    • CSM 可以在广阔的场景中提供高质量的阴影，同时通过只在需要的地方使用高分辨率阴影来优化性能。
    • 它特别适用于大型户外环境，其中远处的物体不需要像近处物体那样细致的阴影细节。
  6. 挑战和限制：
    • 管理和调整不同级联的参数可能比较复杂。
    • 需要精心设计级联的过渡区域，以避免可见的阴影变化。

几何阴影映射（GSM）（TODO）

几何阴影映射（Geometric Shadow Mapping，简称GSM）是一种用于渲染阴影的高级技术，它通过考虑场景中的几何信息来改进阴影的质量。GSM 的主要思想是在阴影映射过程中动态调整阴影贴图的分辨率，以便更有效地利用阴影贴图的分辨率，从而在不同区域提供不同级别的阴影细节。

GSM 的主要特点和工作原理：

  1. 动态适应分辨率：
    • GSM 会根据场景中的几何细节来动态调整阴影贴图的分辨率。在几何复杂或重要的区域（例如，与光源近距离的物体）使用更高的分辨率，而在细节较少或较远的区域使用较低的分辨率。
  2. 几何分析：
    • GSM 通常需要对场景进行一定程度的分析，以确定哪些区域需要更高的阴影分辨率。这可能涉及场景的预处理或实时分析。
  3. 提高性能和质量：
    • GSM 的目标是在保持阴影质量的同时提高渲染性能。通过智能分配资源，它可以减少在不太重要的区域浪费的阴影贴图分辨率。
  4. 实现复杂性：
    • GSM 的实现通常比传统阴影映射更复杂，因为它涉及到场景分析和阴影贴图分辨率的动态调整。
  5. 应用场景：
    • GSM 适用于需要在不同区域精细控制阴影质量的场景，特别是在资源有限但对阴影质量有较高要求的情况下。

GSM 通过结合场景的几何信息和智能的资源分配，提供了一种有效的方法来优化阴影渲染的性能和质量。然而，它的实现可能需要更多的预处理和计算资源，尤其是在实时图形应用中。因此，GSM 的使用需要根据具体的应用场景和性能需求来权衡。

LearnOpenGL15-法线逆矩阵

这行代码中，逆矩阵的应用是为了正确变换模型的法线向量。这个变换的目的是确保法线向量在经过模型变换（比如旋转、缩放等）后仍然正确地表示表面的法线（即垂直于表面的方向）。这里的逆矩阵并不是用来将法线向量转换到某个特定的空间，而是用来正确处理模型变换对法线向量的影响。

  1. 法线变换：当一个模型经历变换（特别是非均匀缩放）时，仅仅用同样的变换矩阵去变换法线向量会导致法线不再垂直于表面。为了解决这个问题，法线向量需要通过模型矩阵的逆转置来变换。
  2. 为什么使用逆转置矩阵：对于法线向量，我们关心的是它们的方向而不是位置。法线向量需要通过模型变换的逆转置矩阵来正确变换，这样可以保证即使在模型被非均匀缩放的情况下，变换后的法线向量仍然垂直于表面。
  3. 空间转换：在这个特定的情况中，法线向量是从模型空间（或局部空间）转换到世界空间。这是因为 model 矩阵将顶点从模型空间变换到世界空间，而法线向量也需要相应地进行变换，以保证它们在世界空间中仍然正确地描述表面的方向。

综上，这行代码的目的是确保模型的法线在经过模型矩阵的变换后，仍然正确地表示模型表面的法线方向，尤其是在模型经历了旋转和非均匀缩放时。

fract(x)

fract 函数在处理周期性或重复性图案时非常有用，特别是在图形学和着色器编程中。

  • 对于正数 x，fract(x) 的结果是 x 本身的小数部分，范围在 [0, 1)。

  • 对于负数 x，fract(x) 仍然返回 [0, 1) 范围内的值，这是因为它返回的是 x 与最接近的较小整数之间的差。

    这种特性方式在纹理映射和图案生成中特别有用，因为它允许在整个 [0, 1) 区间内平滑地处理数值，从而避免了任何潜在的不连续性。在创建重复或平铺的视觉效果时，这一点尤其重要。
